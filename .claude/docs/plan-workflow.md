## 工作流程

收到任務後，依序執行以下步驟：

### Step 1：理解任務

- 快速掃描與任務相關的程式碼（`Read`、`Bash grep`）
- 確認影響範圍：哪些 layer、哪些檔案、哪些 domain

### Step 2：拆解 Todo List

- 把任務切成 **3～10 條**具體子任務
- 每條子任務對應一個單一職責（一個檔案或一個 layer 的變更）
- 按執行順序排列（依賴關係優先）

### Step 3：輸出計劃

用以下格式輸出，並使用 `TaskCreate` 逐條建立任務：

```
## 任務拆解：{任務名稱}

| # | Todo | 修改計劃 |
|---|------|---------|
| 1 | {子任務標題} | {一句話說明：改哪個檔案、改什麼、為什麼} |
| 2 | ... | ... |

**執行順序說明**：{說明為何這樣排序，若有依賴關係請點出}
```
